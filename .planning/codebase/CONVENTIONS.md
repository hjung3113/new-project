# CONVENTIONS - Document-Centered Work

## Session Start

1. Read `.planning/STATE.md`.
2. Read `.planning/ROADMAP.md`.
3. Read the active phase `*-CHECKPOINTS.md`.
4. Read `.scratch/phase-state.json` if it exists.
5. Continue from the named checkpoint or create the next phase context document.

## During Work

- Record durable decisions in `.planning/PROJECT.md` or the active phase context/summary.
- Record adversarial review findings in `*-REVIEW.md`.
- Record exact command evidence in `*-VERIFICATION.md`.
- Update `.planning/STATE.md` whenever the current phase, checkpoint, blocker, or next action changes.

## Phase Close

Before marking a phase complete:

- Summary exists.
- Verification exists.
- Checkpoints are all marked complete or explicitly deferred.
- Success criteria map to evidence.
- `.planning/STATE.md` points to the next action.
