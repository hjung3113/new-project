# Phase 3 Plan 01 - Clean Skeleton and Harness Tooling

## Acceptance Criteria

1. A manifest classifies harness-owned, project-owned, managed, and excluded files.
2. A clean skeleton initializes project-owned planning files without live harness development history.
3. `scripts/harness.py init --target <path>` installs the harness into a target that is not this git repository.
4. `scripts/harness.py upgrade --target <path>` updates harness-owned files while preserving project-owned state and reporting conflicts for locally modified harness files.
5. `scripts/harness.py check` validates JSON, command/mode consistency, clean skeleton contamination, stale phase references, and optional changed-path enforcement.
6. README documents init, upgrade, check, file ownership, and PR-before-merge verification.

## Allowed Paths

- `AGENTS.md`
- `README.md`
- `.planning/`
- `.scratch/phase-state.json`
- `harness/`
- `scripts/harness.py`
- `scripts/test_harness.py`
- `.gitignore`
