# Plan 02-01 - Cache-First DB Context Snapshot

## Acceptance Criteria

1. The snapshot tool defaults to offline cache reads and rejects contradictory `--refresh --offline` usage.
2. Refresh collects fixed MSSQL catalog metadata only and does not execute arbitrary SQL or stored procedures.
3. Process DB drift comparison uses comparable shape fingerprints by default.
4. Refresh can reuse cached per-database detail when table/routine/trigger `modify_date` markers, server/database identity, and redaction/truncation options are unchanged.
5. Secret-bearing generated artifacts are ignored by git by default.
6. Roo workflow skills use `.db-context/` first and return `needs-db-context` when required context is missing or insufficient.
7. The orchestrator handoff contract recognizes `needs-db-context`.
8. Focused Python tests cover no-connect safety, cache reuse, cache non-reuse safety, stale job artifact replacement, redaction, drift comparison, and ignore defaults.
