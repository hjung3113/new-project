## Acceptance Criteria

1. ADR/phase changes have a documented and tested roadmap/state sync invariant.
2. Doctor detects roadmap/state drift and reports severity/cause/impact/fix.
3. Windows users can run DB snapshot refresh using the same Python loader via `.env` or ignored JSON config.
4. DB context snapshot supports full and selected collection for tables, stored procedures, and jobs without connecting in offline mode.
5. Skill and command docs expose the same DB snapshot option vocabulary.
6. Secret-bearing DB context config and generated snapshot artifacts remain gitignored.
7. Doctor reports include diff-before-mutation guidance and do not mutate files.
8. Existing Linux/macOS command paths remain valid.
