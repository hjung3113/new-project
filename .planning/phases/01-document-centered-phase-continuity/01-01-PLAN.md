# Plan 01-01: Add Document-Centered Continuity

## Objective

Create the durable document structure required for phase-by-phase continuation after session reset.

## Tasks

1. Create top-level `.planning` files: `PROJECT.md`, `REQUIREMENTS.md`, `ROADMAP.md`, and `STATE.md`.
2. Create codebase and intel notes that explain the document structure and synthesis.
3. Create Phase 1 folder with context, plan, checkpoints, review, verification, and summary.
4. Update `AGENTS.md` to make `.planning/STATE.md` the first restart point.
5. Update `docs/phase-gate-harness.md` so `.scratch/phase-state.json` links back to `.planning`.
6. Rewrite the temporary handoff file to point at durable docs.

## Acceptance Criteria

- A fresh agent can identify the current phase and next action from `.planning/STATE.md`.
- Phase 1 can be audited from files under `.planning/phases/01-document-centered-phase-continuity/`.
- Future phases have a documented file pattern.
- The handoff file no longer acts as the sole continuity artifact.
